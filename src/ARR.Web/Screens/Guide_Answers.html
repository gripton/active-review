Questionnaire screen- navigation


The questionnaire screen is where you- the session reviewer- answer the specific questions asked by the creator of the session. You use the requirements material as your source for these answers.

The options on the upper navigation bar are:

  • Home- returns you to the home page where you can create, open, and view new and existing sessions.
  • File (menu)- allows you to initiate a actions related to the session you are reviewing such as- open or add to a forum, save, and complete the questionnaire. These are described in detail below.
  • Help on ARR activities- more information and assistance with “answering the questionnaire” (this link)

The question section allows insert answers that you want the creator of the session to read. These responses are based on the requirements specfications provided and your skills and experience with the domain or with softare requirements

  • Edit Answer (edit box)- you use this edit area to insert the answer to the creator's question

You will scroll through the questionnaire questions in using the navigation buttons below the question:

  • Previous- displays the question you had just answered (or viewed). This allows you to scroll back through questions that you may or may not have answered to allow you answer them out of order.
  • Next- displays the next question on the questionnaire. Questions are displayed in the order that the creator set.
  • Complete- activates when you are on the last question and indicates that you have reached the end of the questionnaire. Selecting complete saves the questions and moves the session to the completed state. Remember that you will not be able to return to the questionnaire after it is “completed”.

File menu options are:

  • Forum- takes you to the discussion area of the ARR tool. This is where you can ask questions, give feedback, make comments to the session creator. If you need assistance from the creator this is where you should start. The creator will be notified and, when he/she reads and responds to your comment you will be notified (via email).
  • Save- saves the answers you have entered on the questionnaire screen. To keep the content you have entered you must save before leaving the questionnaire screen. After saving, you can leave then return laterto pick up where you left off.
  • Save and complete- saves the answers you have entered and moves the session to the completed state. This signals the creator that you are finished with the review so assure that you have answer all the questions or have let the creator know why you cannot complete them. Remember that you will not be able to return to the questionnaire after it is “completed”.

Notes on answering the questionnaire- advice for the reviewer

Read through the requirements and questions for high-level understanding

  • Determine if the objectives or goals of the review session are clear- do you have a general idea what the creator of the session is looking for?
  • Ask for specific context for the requirements or questions if background material is not available or content is subjective
  • Initiate a discussion/forum with the creator if you have fundamental questions or don’t understand something about the session

Read individual questions and look for material that provides the answers

  • The objective is to find answers to the questions in this material
  • Assure that you also read comments that creator has added- these should provide background and clarifications to the requirements material

Draft detailed answer using requirements material

Depending on the type the creator is asking, the answers can:

  • Provide a concrete example that uses the material in the requirement specification to form an answer
  • Re-phrase or summarize the requirement to provide assurance that creator’s intent is correctly interpreted
  • State which specific requirements material or specification contains the answer to the question
  • Use the requirements specification to describe how the system should react to a specific condition or “input”

Communicate with the creator if the question is unclear or cannot be answered

  • If you need to assume something to answer the question make the assumption explicit or, better yet, ask the creator what assumption to use
  • The discussion about the question should be done through the ARR tool’s forum feature
  • Forum provides the feedback mechanism that is recorded along with the session questionnaire answers so session context is captured
