Editor screen- navigation


The editor screen is where you- the session creator- can select requirements material, choose a reviewer, create session questionnaire and release the session for review.

The options on the upper navigation bar are:

  • Home- returns you to the home page where you can create, open, and view new and existing sessions.
  • File (menu)- allows you to initiate a number of important actions related to the session you are creating such as- rename, save, preview, release, assign reviewer, spawn. These are described in detail below.
  • Help on ARR activities- more information and assistance with “selecting requirements” (this link), “selecting the reviewer”, and “creating the questionnaire”

The editor page allows you to paste selected material from a software requirement specification into the ARR tool in an organized manner.

  • Edit Requirements (edit box)- you use this edit area to insert the requirements text and images into the editor screen. You can type or cut and paste from another document.
  • Add Requirement (button)- when you have completed your work on a specific requirement, you use this button to actually add the requirement to the list.

For requirements that have been added, you have the option to edit, comment on, or delete.

  • Edit- pulls up a editor window with the current material ready for you to modify in the same way you would when you first edit the requirements.
  • Comment- allows you to make notes or give further descriptions or background on the requirement. These comments are intended to help the reviewer.
  • Delete- removes the requirement and renumbers the remaining ones.

File menu options are:

  • Rename (pop up)- changes the title/name of the review session. Allows you to give the review session a descriptive name so it can be distinguished from other sessions.
  • Save- saves the contents of the session editor screen. To keep the content you have entered you must save before leaving the editor page.
  • Preview- formats the contents of the session editor screen in the same way the reviewer will see the screen once the session is released to him/her. You can check the presentation of the requirements and questions (along with your comments) in this screen.
  • Spawn- allows you to copy specified sections of the review session to a new session. This makes it easy for you to replicate a session or reuse parts of an existing session to create a similar session.
  • Assign reviewer- assigns a reviewer to this session. The session cannot be released until you assign a reviewer. (More information is available in the “selecting the reviewer” section).
  • Release- moves the review session from the create stage to the release stage so the assigned reviewer can start the review process. After a session is released it can no longer be edited by the creator.

Notes on requirements selection- an ideal process

How to select requirements material to present to the reviewer for an active requirements review session:

Determine the objective/goal of the requirements review session

  • What type of review session is this and what is it trying to accomplish (stated succinctly)
  • What does the requirements creator want to confirm or validate
  • What issues or errors is the session seeking to expose
  • What is the scope or coverage of the session both is depth (level of detail) and breadth (areas of the specification session will cover)
  • What information is need to start this session and what is the desired end result for this session
  • Where does this session fit in with respect to other previously conducted or upcoming planned review sessions- where does it overlap, complement, enhance, other sessions

Assemble all available requirements material

  • Collect the material in the form of documents, diagrams, prototypes (if any) available for use in this session
  • Assess if the documentation available covers the scope of the review (the specific topic and level of detail)
  • Create supplementary material to provide context for this session if needed
  • Reformat material for easy of understanding and review if this has not been done

Formulate review questions in rough outline form

  • Consider, in general, the questions that the creator will ask the reviewer for this session
  • Create classes of questions that relate to the session goal
  • Consider specific methods for exposing issues we are looking for in the session
  • Determine questions at the boundary of the session’s scope (what is in versus out of scope for the session)

Determine your answers to the proposed questions

  • Using all of the requirement material assembled above, determine what you need to answer classes of questions
  • Consider how much or what level of requirement specification detail is needed to answer the questions
  • Use the boundary questions to determine the complete range of material needed to answer the questions

Assemble the material you needed to answer the questions

  • Assemble the material you needed to answer the questions
  • Format the extracted material for ease of reference and review
