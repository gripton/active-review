Home page- navigation


  • Create new session (button)- starts the ARR session create process. This action moves the review session to the “create” state so then it is visible in the “My New Created Sessions” list. This will take you to the session editor screen.
  • My New Created Sessions (list)- contains the creator’s sessions that have been started but are not released. As a session creator, you populate the session (with requirement material and ARR questionnaire) in the editor screen and assign the reviewer. You can then release a session (from the session editor screen) when you are ready.
  • My Active Review Sessions (Reviewer) (list)- contains sessions that have been assigned to you as a reviewer. The session creator selects the reviewer for a session. You can view the requirements material and ARR questionnaire (from the questionnaire reader screen).
  • My Active Review Sessions (Creator) (list)- contains sessions that you, as the creator, have created and, then, released. You have assigned a reviewer to each of these sessions so they will see the session in their “Active Review Session (Reviewer)” list.
  • My Archived Sessions (list)- contains the sessions that the reviewer has completed and the session creator has assessed/checked and archived for future reference.

Roles


  • Creator- the author of requirements specification or individual responsible for requirements analysis and verification. Must have intimate familiarity with a project’s requirements and must have the background to understand the intent of the requirements and the project. Designs the review session and develops its objectives. Is motivated to create the active review in order to determine whether the requirements specification has met specific criteria and can be used to successfully create all downstream software artifacts.
  • Reviewer- individual with specific skills that can be used in the requirements review process. The skill included domain familiarity (as a user or customer of a product), general requirements review and validation experience, ability to determine qualities of documents, and other related skills. Is motivated to participate in the active review to share his/her skills with the broader team and to make a positive contribution to the quality of the development project.

Tasks


  • Prepare requirements specification for review session- determine the parts of the requirements specification or other material needed to focus the reviewers attention on the specific concerns of the review session. This should be all of the material that the reviewer needs to answer the questions that the creator posses. Provide context for the requirements material if it is needed. Organize and insert this material into the session review template.
  • Create review questionnaire for review session- develop a a set of questions about the requirements under review in the session that will allow the creator to explore the qualities of the requirement. The reviewer should be able to use the material provided by the creator in the review session template to give detailed, descriptive answers to the questions beyond a simple “yes/no” answer.
  • Assign reviewer for review session- find the best reviewer to focus on the concerns of the review session based based on the skills of the set of possible reviewers and objective of the review session. Select this reviewer for the session. Reviewer will be notified when the session is released. (If reviewer does not have an account in the system, invite the him/her to create an account.)
  • Release review session- upon completion of the review session creation tasks above, release the session to the reviewer so he/she can gain access to the specification material and the questions and start their part of the review process.
  • Answer review session questionnaire- the reviewer provides detailed answers to the questions in the session questionnaire after a thorough review of the requirements material. He/she should directly use the material to come up with deep and well thought out answers to the questions. When the reviewer is done answering the questions, he/she selects save and complete to notify the creator the questionnaire is complete.
  • Discuss review questions and requirements in a forum- if the reviewer is unclear on the session objectives, material under review or has questions, he/she initiates a forum-style discussion with the creator. This also applies to the creator of he/she feels the need to clarify the review session content or the questionnaire in any way.
  • Archive review session- after all questions are answered, the reviewer has selected complete, and the creator has clarified the answers (if necessary), store the contents of the review session (requirements, questionnaire questions and answers, forum material) if a form where it can be accessed for future reference. Session material will be read only after it is archived.

Products


  • Requirements specification excerpt- textual material from the software requirements specification and possibly other sources that the creator wants to be reviewed by the active review process.
  • Review session questionnaire questions- specific questions that the creator wants the review to answer using the material provided.
  • Review session questionnaire answers- specific answers (as “positive assertions”) to the review questions provided by the creator.
  • Review session forum discussion- record of the interactions between the creator and reviewer if either one need to clarify a question, requirement or other topic in the active review process. (Provides context to the questionnaire questions and answers.)
